# 리액트 이론
# 코딩 -> 실행(dev - frontend) -> 결과확인(웹브라우저)
# jsp vs 리액트
# jsp 동작 : 사용자 -> 웹브라우저 -> 메인(html) 요청 -> 벡엔드(db 정보 + html 만듬)
                  -> 다시 웹브라우저로 html 전송 (SSR: Server Side Request, 벡엔드가 html 만듬) 
# 리액트 동작: 사용자 -> 웹브라우저 -> 메인(html) 요청 -> 프론트(html 일부만 만들기)
                  -> 벡엔드:정보만 전송(그릇,접시) -> 리액트: html 안에 정보를 넣어서 완성
                  (CSR: Client Side Request, 프론트가 html 만듬)
# 장점: jsp (구글 등에 잘 노출됨, 광고효과 업)
       리액트(속도가 빠르다)
# 단점: jsp(번쩍거림, 새로고침 현상이 빈번)
       리액트(구글 등에 덜 노출됨 -> 다른 방법으로 개선)

# jsp(바닐라,jquery): html의 태그를 찾아서 색깔 등을 바꾸기
# 동작: 1) 태그를 찾기 : 10000라인
#      2) 비교 : 개별적 비교
#      3) html 파일: 현재 html만 저장 

# 리액트      : html 파일( 메모리 저장 ): 과거 html (DOM)
#                                   현재 html (작성중, 수정중, DOM)
#             : 과거 html vs 현재 html (전체 비교, 틀린부분만 화면에 반영)
