#       4) 인터페이스(객체 별명,types 폴더, 1번만) -> 5) 서비스(벡엔드 통신 함수) ->
#       6) 빈페이지.tsx(js 코딩, html 코딩 추가)


# 샘플 활용 코딩: (실무 기법)
# 부서(난이도 낮음)   -> 예) qna, faq, 공지사항, 자유게시판(답변글 제외) 등
# FileDb -> 예) 쇼핑몰 메인화면(이미지, 제목), 첨부파일 게시판, 갤러리 게시판 등 

# 난이도 높음: 장바구니, 결재, 로그인화면, 회원가입, 소셜로그인, 예약 게시판

# 이론) 리액트 아키텍쳐
# 특징) 1) SPA(Single Page Application): html 을 1개 사용하는 프로그램(리액트,뷰,앵귤러)
#     vs (비교): MPA(Multiple Page Application): html 을 여러개 사용하는 프로그램(JSP, html)
#       2) 바인딩 기술: 상태함수, 상태변수를 사용해서 화면 갱신을 빠르게하는 기술
#       3) Virtual DOM: 이전 html 구조를 저장하고 있어서 변경된것을 빠르게 수정가능합니다.

# 인증/권한관리: 
# 1) 로그인: 
#    (1) 로그인 화면 입력 -> 벡엔드 처리(쿠키:웹토큰 넣음)
#    (2) 벡엔트 통신 플로그인 설정파일(axios) : httpOnly(js 편집 막기 옵션) 옵션 추가
     => 수정: CommonService.ts 
#    (3) 공유저장소 플로그인 사용: 로그인 유무를 저장(js: 새로고침하면 값이 사라짐)
#    (4) 새로고침 대응 코드      : 벡엔드에 내가 로그인 유저 있니 한번더 물어봄
                                  -> 다시 로그인 유무를 재저장
     => App.tsx 에 추가(새로 고침시 다시 이 페이지 읽음)
